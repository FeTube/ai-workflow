主表从表关系关联字段逻辑说明
t_user t_oauth_provider 1 : N t_user.id → t_oauth_provider.user_id 一个用户可绑定多个第三方账号（Google、Apple 等），每个 provider+provider_id 组合唯一
t_user t_password_reset_token 1 : N t_user.id → t_password_reset_token.user_id 用户每次发起「忘记密码」生成一条令牌记录，令牌一次性使用，过期 24 小时
t_user t_login_attempt 1 : 1 t_user.id → t_login_attempt.user_id (UK) 每用户一行，记录连续失败次数；≥5 次触发 30 分钟锁定；登录成功后 fail_count 重置
t_user t_agreement_consent 1 : N t_user.id → t_agreement_consent.user_id 注册时必须同意 ToS + Privacy Policy；协议版本更新时需用户重新同意，保留历史记录用于合规审计
t_user t_file 1 : N t_user.id → t_file.user_id 用户上传的头像、附件等文件记录；file_id (UUID) 对外暴露，避免自增 ID 泄露
t_user Redis 缓存 逻辑 t_user.email → verify:*:{email}
t_user.id → token:user_invalidated:{userId}		 注册/重置密码流程通过 email 关联验证码缓存；修改密码时写入 user_invalidated 使旧 Token 全部失效
⚡缓存模型 — Redis Key（7）
verify:{purpose}:{email}
6 位验证码，purpose = register | reset_password
类型: string TTL: 10 min
verify:attempts:{purpose}:{email}
验证码校验尝试次数，max 5
类型: string (counter) TTL: 10 min
verify:cooldown:{email}
发送冷却锁，60 秒内不可重发
类型: string TTL: 60 sec
verify:hourly:{email}
每小时发送计数，max 5
类型: string (counter) TTL: 1 hour
pending_reg:{email}
待验证注册信息（password hash + agreeToTerms）
类型: string (JSON) TTL: 15 min
token:blacklist:{jwt}
已注销 Token 黑名单
类型: string TTL: token 剩余有效期
token:user_invalidated:{userId}
用户所有 Token 失效时间戳（密码修改触发）
类型: string (timestamp) TTL: 30 days

🔒安全方案

认证

JWT (access 2h + refresh 30d), HttpOnly Secure Cookie option

密码

BCrypt encoding, min 8 chars, uppercase+lowercase+digit

OAuth

Authorization code flow, server-side token exchange, state parameter CSRF protection

Cookie

HttpOnly, Secure, SameSite=Lax, Domain=.looply.com
