!冲突与差异（7）
命名不一致
命名不一致：PRD 中引用的是 Screen/ErrorState 和 H5/ErrorState，但设计稿中实际使用的是 Screen/LoginError 和 H5/LoginError。建议统一为设计稿命名（LoginError），并同步更新 PRD。
警告
设计多余
设计稿中存在 Screen/Login-Remembered 页面（回访用户登录变体，显示头像、用户名和仅密码输入），但 PRD 中未将其定义为独立页面。PRD 3.6 节提到了 Remember Me 功能但未定义此 UI 变体。建议将该页面补入 PRD 或确认其归属 F-AUTH-014。
警告
交互不一致
忘记密码按钮文案不一致：设计稿显示「Send reset link」，但 PRD 3.4 节明确发送的是验证码而非重置链接（"Send Reset Code"）。实际机制为验证码方式，设计稿按钮文案应更新为「Send reset code」。
警告
缺失页面
H5 端缺少 Login-Remembered 变体页面。PC 端有 Screen/Login-Remembered 但 H5 没有对应的 H5/Login-Remembered。如果确认 Login-Remembered 为正式需求，需补齐 H5 端设计稿。
提示
需求模糊
PRD 2.5 节要求展示密码强度指示器（弱/中/强，颜色区分），但 PC 端和 H5 端设计稿中均未包含密码强度组件。需确认是延后实现还是需要补充设计。
警告
需求模糊
PRD 提到首页作为登录/注册的统一入口以及退出后的跳转目标，但尚无首页设计稿（PRD 注明「首页尚未设计」）。这是已知的待补充项，留待后续迭代。
提示
需求模糊
设计稿在注册页和登录页（PC + H5）中包含国家/地区选择器组件（国旗 + 国家名 + 下拉箭头），但 PRD 中未提及国家/语言切换功能。需确认该组件是未来 i18n 功能的占位还是纯装饰。
