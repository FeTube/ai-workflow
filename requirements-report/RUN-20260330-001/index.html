!冲突与差异（8）
缺失页面
所有 H5 端页面均标注「待设计」。PRD 1.2 节明确适用终端包含 H5 移动端，但 .pen 设计稿仅包含 PC 端页面，H5 端 0 个设计页。需补齐 H5 端设计稿。
严重
缺失页面
PRD 5.1 节「密码修改（登录态）」功能无对应设计稿，PRD 中标注「待设计（账号设置页内）」。需补齐密码修改页设计。
警告
缺失页面
PRD 5.4 节「协议版本检测」弹窗无对应设计稿，PRD 中标注「待设计（弹窗形式）」。需补齐协议更新弹窗设计。
警告
交互不一致
PRD 2.5 节密码规则提示为「At least 8 characters with a mix of letters, numbers, and symbols」，含 symbols。但 PRD 6.3 节密码校验规则仅要求大小写字母+数字，symbols 为可选。设计稿 Screen/Register 密码提示文案需与最终规则一致。
警告
需求模糊
PRD 提到「首页尚未设计」，退出登录、Session 过期等多个流程的跳转目标为「首页」。首页的具体形态和入口布局未定义，可能影响注册/登录入口的实际实现。
提示
需求模糊
产品架构图 v8.4 包含「手机号注册」「验证码登录（手机号/邮箱）」「Passkey 登录」等能力，但 PRD V2.0 MVP 未覆盖这些功能。需明确这些是后续迭代范围还是遗漏。
提示
命名不一致
PRD 2.9 节标题为「价值点展示」，但该节应为 2.9 而非 2.7 编号后的 OAuth 补充信息页序号。PRD 2.7 已用于 OAuthSupplementary，2.9 用于价值点展示。编号跳过了 2.8（已用于验证码），实际无编号错误，仅顺序稍显跳跃。
提示
缺失页面
PRD 4.3 节退出入口描述了「用户头像下拉菜单」交互，但设计稿中无用户头像下拉菜单的设计。Screen/LogoutModal 仅为弹窗确认，触发入口未设计。
